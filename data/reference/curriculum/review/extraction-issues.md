# Extraction Issues

## 현재 데이터 상태

1. 자동 파서 기반 정규화는 현재 보류 상태다.
2. 현재 저장소의 `canonical/`은 승인 데이터가 아니라 빈 템플릿 상태를 유지한다.
3. 구조 예시는 `templates/`에만 두고, 실제 데이터는 검수 기준 정리 후 재등록한다.

## 현재 보류 항목

1. PDF의 인쇄 페이지 번호와 실제 PDF 페이지 인덱스가 다르다.
2. 학과 안내 본문이 2단 레이아웃이라 `pdftotext` 기본 추출만으로는 블록 경계가 자주 섞인다.
3. 상경·사회 계열처럼 한 페이지에 여러 학과가 병렬 배치된 구간은 학과별 자동 분리가 안정적이지 않다.

## 다음 검수 우선순위

1. 수동 입력 기준 필드와 최소 필수 항목 확정
2. `templates/` 기반 검수용 입력 워크플로우 정의
3. 승인 가능한 단위부터 소량 수동 등록
4. 필요 시 자동 파싱은 review 보조 도구로만 재검토
