---

## 1. 목적

본 문서는 `학생부 Insight` 화면의 정보 구조와 상호작용을 구현 가능한 수준으로 정의한다.

MVP 목표:

1. 학생이 관심 기반 피드를 빠르게 소비할 수 있게 한다.
2. 학생이 긴 글 없이도 저장과 메모를 남길 수 있게 한다.
3. 학생이 저장한 항목을 탐구 노트처럼 다시 볼 수 있게 한다.

---

## 2. 전체 화면 구조

화면은 세로 스크롤 단일 페이지 구조를 기본으로 한다.

상단부터 아래 순서로 배치한다.

1. 헤더
2. 소개 히어로
3. 추천 키워드 요약
4. 피드 탭
5. 피드 목록
6. 저장한 탐구 노트
7. 후속 액션 영역

---

## 3. 화면 블록 정의

## 3.1 헤더

구성:

1. 뒤로가기 버튼
2. 제목 `학생부 Insight`
3. 보조 액션 영역

보조 문구:

`관심 직업, 관심 학과와 연결된 최신 이야기를 탐색하고 탐구 노트로 저장하세요.`

## 3.2 히어로 카드

구성:

1. 강조 문장
2. 서브 문장
3. 주요 CTA

문구 예시:

- 제목: `최신 이슈를 학생부 소재로 바꾸세요`
- 설명: `관심 직업과 학과에 맞는 이야기를 보고, 메모하고, 탐구 주제로 이어갈 수 있습니다.`
- CTA: `관심 기반 피드 보기`

## 3.3 추천 키워드 요약

구성:

1. `관심 직업`
2. `관심 학과`
3. `최근 기록 기반 주제`

표현:

1. 칩 형태
2. 최대 5개 노출
3. 부족 시 빈 상태 문구 제공

빈 상태:

`관심 직업과 학과를 저장하면 더 잘 맞는 피드를 추천해드려요.`

## 3.4 피드 탭

탭 목록:

1. `전체`
2. `관심 직업`
3. `관심 학과`
4. `학생부 연결`

동작:

1. 탭 전환 시 피드 재조회
2. 선택 탭 강조
3. 탭 전환은 페이지 이동 없이 처리

## 3.5 피드 카드

카드 구성:

1. 제목
2. 출처/발행일 메타
3. 2~3줄 요약
4. `왜 중요한가`
5. 태그 영역
6. 학생부 연결 포인트
7. 탐구 질문
8. 액션 버튼

액션 버튼:

1. `저장`
2. `메모`
3. `원문 보기`

MVP 규칙:

1. 카드 내 텍스트는 과도하게 길지 않게 절단
2. 태그는 4개 이내 우선 노출
3. 학생부 연결 포인트는 2개 이내 우선 노출

## 3.6 메모 입력 패널

형태:

1. 카드 확장형 인라인 입력 또는 하단 시트

입력 요소:

1. 짧은 메모 textarea
2. 연결 직업/학과 표시
3. 저장 버튼
4. 취소 버튼

플레이스홀더 예시:

`왜 흥미로웠는지, 어떤 과목이나 활동과 연결되는지 짧게 남겨보세요.`

제약:

1. 300자 이내
2. 미입력 상태에서도 단순 저장 가능

## 3.7 저장한 탐구 노트

구성:

1. 섹션 제목
2. 최근 저장 항목 리스트
3. 상태 뱃지
4. 메모 미리보기

항목 구성:

1. 제목
2. 저장일
3. 메모 요약
4. 연결 직업/학과
5. 보조 액션

보조 액션:

1. `메모 수정`
2. `삭제`
3. `기록으로 연결`

## 3.8 후속 액션 영역

MVP에서는 안내형 카드로 시작한다.

카드 예시:

1. `탐구 보고서 아이디어로 이어보기`
2. `학생부 기록 페이지에서 정리하기`

CTA:

1. `/records/new?intent=careerIdea`
2. `/records`

## 4. 상태별 화면 정의

## 4.1 로딩 상태

1. 키워드 칩 스켈레톤
2. 피드 카드 스켈레톤 3개
3. 저장 목록 스켈레톤 2개

## 4.2 빈 상태

조건:

1. 관심 직업/학과가 없고 피드도 비어 있음

노출:

1. 빈 상태 문구
2. 관심 직업/학과 저장 유도 CTA

문구 예시:

`아직 맞춤 피드가 준비되지 않았어요. 관심 직업과 학과를 저장하면 학생부 Insight가 더 정확해집니다.`

## 4.3 에러 상태

1. 피드 조회 실패 시 섹션형 경고 메시지
2. 저장 목록 실패 시 별도 경고 메시지
3. 재시도 버튼 제공

## 4.4 부분 성공 상태

조건:

1. 외부 피드 수집 실패
2. fallback 콘텐츠로 대체

노출:

`일부 최신 콘텐츠를 불러오지 못해 기본 추천 콘텐츠를 먼저 보여드리고 있어요.`

---

## 5. 모바일 우선 레이아웃

## 5.1 기본 원칙

1. 모바일 단일 컬럼
2. 카드 간 간격 충분히 확보
3. 버튼은 한 손 조작 가능한 크기 유지

## 5.2 데스크톱 확장

1. 히어로와 저장한 탐구 노트는 2열 가능
2. 피드 목록은 단일 컬럼 유지 가능
3. 탭과 키워드 칩은 가로 공간 활용

---

## 6. 인터랙션 규칙

## 6.1 저장

1. 저장 버튼 클릭 시 즉시 상태 변경
2. 성공 시 `탐구 노트에 저장됨` 토스트 또는 인라인 상태 표시
3. 이미 저장된 경우 `저장됨` 상태 유지

## 6.2 메모

1. 메모 열기
2. 입력
3. 저장
4. 성공 시 카드와 저장 목록에 즉시 반영

## 6.3 원문 보기

1. 새 탭으로 열기
2. 출처 이동 전 사용자 컨텍스트 손실 방지

## 6.4 기록 연결

MVP에서는 아래 방식으로 제한한다.

1. `기록으로 연결` 클릭 시 학생부 기록 페이지 또는 새 기록 페이지로 이동
2. 후속 버전에서 저장 항목 ID를 쿼리로 전달해 프리필 가능

## 9. 공개 사례 참고

화면 패턴 참고:

1. `Refeed`
  - 피드 카드와 저장 흐름의 단순한 상호작용 패턴 참고
2. `jotty`
  - 메모 축적과 초안 작업 분리 패턴 참고

적용 원칙:

1. 학생 대상 서비스인 만큼 정보 밀도는 높되 입력 부담은 낮춘다.
2. 일반 북마크 UI가 아니라 `탐구 노트` 성격이 느껴지도록 라벨을 조정한다.
